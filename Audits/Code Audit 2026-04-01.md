# Аудит коду — 2026-04-01

## Загальна оцінка

Добре структурований MVP з правильним async/sync bridge, compile-time SQL, `rust_decimal` для сум та state machine для статусів актів. Нижче — знайдені проблеми за пріоритетом.

---

## Критичні

### ✅ SQL: рік порівнюється як f64 (`acts.rs:38`)
**Проблема:** `EXTRACT(YEAR FROM date) = $1` з `year as f64` — крихкий implicit cast.  
**Виправлення:** `EXTRACT(YEAR FROM date)::int = $1` + `year as i32`  
**Статус:** Виправлено

---

## Середні

### ✅ Відсутня валідація полів форми перед збереженням (`main.rs`)
**Проблема:** Порожній номер, дата або невибраний контрагент могли пройти до БД без перевірки.  
**Виправлення:** Додано явну валідацію в `spawn_save_act` і `on_act_form_update` до парсингу.  
**Статус:** Виправлено

### ⏳ Редагування позицій акту не реалізовано (`act_form.slint`, `main.rs`)
**Проблема:** Після "Додати позицію" всі поля `0.00` — inline-редагування відсутнє.  
**Виправлення:** Потребує `on_act_form_item_changed(idx, field, value)` callback + перерахунок total.  
**Статус:** TODO (окреме завдання)

### ⏳ Помилки async операцій — тихі (немає UI feedback)
**Проблема:** Якщо save/update/status change не вдались — тільки `tracing::error!` в консоль, користувач нічого не бачить.  
**Виправлення:** Потребує toast/snackbar компонент в Slint + `error-message` property на MainWindow.  
**Статус:** TODO (окреме завдання)

### ⏳ Typst — зовнішня CLI без graceful degradation
**Проблема:** PDF кнопка повністю зламана якщо `typst` не встановлено, без зрозумілого повідомлення для користувача.  
**Виправлення:** Перевіряти наявність Typst при старті; якщо відсутній — показувати попередження в UI.  
**Статус:** TODO (PDF генерація ще не підключена до UI)

## Низькі

### ⏳ Відносний шлях `storage/` залежить від CWD
**Проблема:** `PathBuf::from("storage/documents/acts")` — поведінка залежить від директорії запуску.  
**Виправлення:** Використовувати `%APPDATA%/Acta/` або шлях відносно exe-файлу.  
**Статус:** TODO

### ⏳ Тести лише для PDF генератора
**Проблема:** `generator.rs` має unit-тести для `amount_to_words`; `db/acts.rs` — жодного.  
**Виправлення:** Integration tests для `create`, `update_with_items`, `advance_status`.  
**Статус:** TODO
