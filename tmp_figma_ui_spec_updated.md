## Ключові оновлення по структурі макету

### Sidebar

У поточному Figma sidebar містить:
- `Головна`
- `Акти`
- `Накладні`
- `Контрагенти`
- `Платежі`
- `Звіти`
- `To-Do`
- `Календар платежів`
- `Налаштування`

Отже, старий ASCII-опис sidebar у попередній версії spec був неповним.

### Акти

Поточний список актів (`163:2`) має таку структуру:
- Topbar із заголовком `Акти` і кнопкою `+ Новий акт`
- Статусні вкладки: `Всі`, `Чернетки`, `Виставлені`, `Підписані`, `Оплачені`
- Рядок пошуку, кнопка фільтра, вибір місяця
- Таблиця актів
- Пагінація

Поточна форма акту (`163:242`) містить:
- Breadcrumb `Акти / Новий акт`
- Основну форму
- Таблицю позицій
- Підсумки `Разом` і `До сплати`
- Праву панель `Статус акту`
- Окрему картку `Підказка`
- Поле `Примітки`

Це означає, що стара секція spec про форму акту вже частково застаріла: у фактичному макеті є більше елементів, ніж було зафіксовано раніше.

### Контрагенти

Для блоку `Контрагенти` підтверджено актуальний canvas `84:3`. Усередині нього точно існують:
- `103:2` — новіший список контрагентів;
- `91:198` — картка контрагента;
- також у файлі лишаються старіші валідні вузли на кшталт `3:3` і `18:64`.

Отже, попередня версія spec по змісту була близька до реальності, але не розрізняла актуальніші та legacy-вузли.

Блок контрагентів у Figma в цілому відповідає попередньому опису:
- список контрагентів;
- форма;
- картка контрагента;
- company picker;
- dropdown перемикача компаній;
- management grid.

## Що ще треба дозвірити окремо

Під час цієї звірки не були надійно зафіксовані окремі актуальні `node-id` для:
- Dashboard / `Головна`
- Платежі
- Накладні
- Звіти

Після додаткового вибіркового обходу top-level вузлів Figma ці екрани все ще не вдалося підтвердити як окремі актуальні frame/canvas у доступному дереві. Тому для них у документі свідомо лишено `TBD`, а не випадкові або застарілі значення.

Тому для цих екранів у цьому документі поки лишено `TBD`, щоб не закріплювати недостовірні значення.

## Висновок

Попередня версія `Figma UI Spec.md` добре описувала загальний задум, але вже не була надійною як технічне джерело для `node-id`.

Після оновлення в цьому файлі:
- виправлено актуальні `node-id` для `Акти`;
- зафіксовано, що `Контрагенти` й компанії лишаються валідними;
- відмічено застарілі посилання;
- очищено документ від неактуальних прив'язок, які могли збивати під час реалізації.
